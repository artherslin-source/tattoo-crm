# Cursor Deployment Policy for Tattoo-CRM

🚫 禁止直接透過 `railway up` 或 CLI 命令部署任何分支。
✅ 所有部署必須依下列流程進行：

1. Cursor 先在本地執行 `git add . && git commit -m "..." && git push`
2. GitHub Action 或 Railway webhook 自動偵測 push → 自動建置部署
3. Cursor 不得自行執行 `railway up`、`railway deploy` 或 CLI 直接部署命令

### Environments
- `main` → Production
- `staging` → Staging (for QA & client testing)

### Notes
- Commit message 必須包含環境代碼，例如：
  - `feat(staging): update appointment UI`
  - `fix(prod): correct API base url`
- Cursor 在任何 build、debug、修復任務中均需使用 GitHub Push 流程。
