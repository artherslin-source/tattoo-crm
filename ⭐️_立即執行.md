## 🔍 部署後驗證

### 1. 獲取 URLs

```bash
# 後端 URL
cd backend && railway domain

# 前端 URL
cd frontend && railway domain
```

### 2. 測試後端

```bash
# 替換成實際的後端 URL
curl https://你的後端URL/health

# 預期回應
{"status":"ok","timestamp":"..."}
```

### 3. 測試前端

開啟瀏覽器前往前端 URL，按 F12 查看 Console：

```
🔍 API Base URL: https://...
🔍 Environment: staging
```

### 4. 更新 CORS 和 API URL（如需要）

獲得 URLs 後：

```bash
# 更新後端 CORS
cd backend
railway variables --set "CORS_ORIGIN=https://你的前端URL"
railway up --detach

# 更新前端 API URL
cd ../frontend
railway variables --set "NEXT_PUBLIC_API_BASE_URL=https://你的後端URL"
railway up --detach
```

## 🆘 遇到問題？

### DATABASE_URL 未設置？

在 Railway Dashboard 中：
1. 前往 `tattoo-crm-backend-staging` 專案
2. 點擊 "New" → "Database" → "Add PostgreSQL"
3. Railway 會自動設置 `DATABASE_URL`
4. 重新部署：`railway up --detach`

### CORS 錯誤？

```bash
cd backend
railway variables --set "CORS_ORIGIN=https://完整的前端URL"
railway up --detach
```

### 前端顯示 localhost:4000？

```bash
cd frontend
railway variables --set "NEXT_PUBLIC_API_BASE_URL=https://完整的後端URL"
railway up --detach  # 必須重新部署
```
